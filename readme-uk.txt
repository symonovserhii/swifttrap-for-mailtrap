Надсилайте листи WordPress через Email API Mailtrap (не SMTP). Масові та транзакційні потоки, категорії, список блокувань, журнал листів.

== Description ==

**SwiftTrap** — це пряма заміна `wp_mail()`, яка спрямовує пошту WordPress через **Mailtrap Email Sending API** замість SMTP. Плагін створено спеціально для Mailtrap — це не універсальний SMTP-плагін із пресетом Mailtrap, — тож він відкриває доступ до нативних можливостей Mailtrap, недоступних через SMTP: маршрутизацію між масовим і транзакційним потоками, категорії листів, власні змінні для трекінгу, списки блокувань і статус перевірки домену.

= Чому HTTP API, а не SMTP? =

* **Менша затримка** — один HTTPS-виклик на повідомлення, без циклів MAIL FROM / RCPT TO / DATA.
* **Краща доставленість** — Mailtrap спрямовує повідомлення API через власні виділені транзакційний і масовий потоки; SMTP не дає можливості вибору потоку.
* **Нативні категорії** — кожен лист автоматично категоризується (welcome, password-reset, notification, marketing тощо), тож ви можете фільтрувати їх і будувати звіти в Mailtrap.
* **Без проблем із фаєрволом** — порти 587/465 заблоковано? API працює через стандартний HTTPS 443.

= Чому SwiftTrap, а не WP Mail SMTP / Post SMTP =

* Універсальні SMTP-плагіни використовують SMTP-облікові дані Mailtrap і втрачають усі функції, доступні лише через Mailtrap.
* SwiftTrap звертається до `send.api.mailtrap.io` для транзакційної пошти та до `bulk.api.mailtrap.io` для масової — автоматично, на основі категорії або через фільтр.
* Не потребує PHP SDK Mailtrap. Плагін важить **приблизно 30 КБ загалом** і використовує лише HTTP API WordPress (`wp_remote_post`).
* Сторінка статистики показує статус перевірки вашого домену відправки та актуальний список блокувань (відмови, скарги, відписки).

= Можливості =

* Пряма заміна `wp_mail()` — працює з WooCommerce, Contact Form 7, Gravity Forms та будь-яким плагіном, що використовує пошту WordPress.
* Автоматична категоризація листів і перевизначення маршрутизації потоків через таблицю налаштувань.
* Відстеження доставки та вебхуки — трекінг подій у реальному часі через власний REST-маршрут `swifttrap/v1/webhook`.
* Управління блокуваннями — панель CRUD для списків блокувань Mailtrap із перевіркою отримувачів на блокування перед відправкою.
* Надійний резервний варіант — плавне перемикання на нативний `wp_mail()` WordPress, якщо виклик API Mailtrap не вдається.
* Інтеграція зі Site Health — тест перевірки статусу токена Mailtrap і верифікації домену відправки.
* Живий журнал листів — перегляд і фільтрація даних доставки, отриманих напряму з API Mailtrap; пошук за адресою отримувача, статусом або діапазоном дат з автоматичною пагінацією.
* Команди WP-CLI — керування з командного рядка через `wp swifttrap` (test, stats, prune-logs, send-suppression-sync).
* Захист розміру вкладень — налаштовувані ліміти, що запобігають відхиленню завеликих файлів на шлюзі API.
* Кнопка тестового листа на сторінці налаштувань.
* Підтримка шаблонів Mailtrap через `template_uuid`.
* Повертається до стандартного обробника пошти WordPress, якщо плагін вимкнено або токен порожній.

= Розширюється через фільтри =

* `swifttrap_mailtrap_email_category` — перевизначає автоматично визначену категорію листа.
* `swifttrap_mailtrap_use_bulk_stream` — примусово спрямовує повідомлення в масовий або транзакційний потік.
* `swifttrap_mailtrap_template` — надсилає через шаблон Mailtrap за `template_uuid`.
* `swifttrap_mailtrap_custom_variables` — додає метадані трекінгу до вихідних листів.

= Конфіденційність =

Цей плагін надсилає вміст листів (отримувачі, тема, тіло, вкладення) до API Mailtrap за адресами `send.api.mailtrap.io` та `bulk.api.mailtrap.io`. Статистика облікового запису та журнал листів отримуються з `mailtrap.io/api/accounts` і `mailtrap.io/api/email_logs`. Див. [Політику конфіденційності Mailtrap](https://mailtrap.io/privacy-policy). Жодні дані нікуди більше не надсилаються.

== Installation ==

1. Встановіть через **Плагіни → Додати новий** і знайдіть *SwiftTrap for Mailtrap*, або завантажте папку `swifttrap-for-mailtrap` до `/wp-content/plugins/`.
2. Активуйте плагін.
3. Перейдіть до **Mailtrap → Налаштування**.
4. Вставте свій **Send API token** Mailtrap (панель Mailtrap → Sending Domains → API Tokens).
5. Вкажіть свій перевірений email і ім'я відправника.
6. Натисніть **Надіслати тестовий лист**, щоб перевірити доставку.

== Frequently Asked Questions ==

= Чому варто використовувати SwiftTrap замість WP Mail SMTP чи Post SMTP з обліковими даними Mailtrap? =

WP Mail SMTP та Post SMTP працюють через SMTP-шлюз Mailtrap і сприймають Mailtrap просто як ще один SMTP-хост. SwiftTrap використовує HTTP Send API Mailtrap, який відкриває можливості, недоступні через SMTP: маршрутизацію між масовим і транзакційним потоками, категорії, власні змінні трекінгу, UUID шаблонів і живий перегляд списку блокувань. Обирайте SwiftTrap, якщо хочете нативної поведінки Mailtrap; обирайте універсальний SMTP-плагін, якщо потрібне одне налаштування для всіх провайдерів.

= Чи підтримує він шаблони листів Mailtrap? =

Так — використовуйте фільтр `swifttrap_mailtrap_template`, щоб надсилати через `template_uuid`. Змінні шаблону можна передавати через стандартний набір template-variables Mailtrap.

= Як працює маршрутизація масового потоку? =

За замовчуванням маркетингові/рекламні категорії спрямовуються до `bulk.api.mailtrap.io`, а все інше — до `send.api.mailtrap.io`. Перевизначайте це для окремого повідомлення за допомогою фільтра `swifttrap_mailtrap_use_bulk_stream` — корисно для масових розсилок зі стороннього плагіна.

= Де отримати API-токен? =

Увійдіть на [mailtrap.io](https://mailtrap.io), відкрийте свій домен відправки, перейдіть до **API Tokens** і створіть токен із дозволами на відправку.

= Що станеться, якщо я вимкну плагін або видалю токен? =

WordPress повертається до свого стандартного обробника `wp_mail()`. Жоден лист не втрачається непомітно.

= Чи потребує плагін PHP SDK Mailtrap? =

Ні. SwiftTrap звертається до REST API Mailtrap напряму через HTTP API WordPress. Загальний розмір плагіна — близько 30 КБ.

= Які дані передаються назовні? =

Дані листів (отримувачі, тема, тіло, вкладення) передаються до `send.api.mailtrap.io` та `bulk.api.mailtrap.io`. Статистика облікового запису отримується з `mailtrap.io/api/accounts`. Див. [Політику конфіденційності Mailtrap](https://mailtrap.io/privacy-policy).

= Чи є обмеження на розмір вкладень? =

Так — 25 МБ на лист (відповідає ліміту API Mailtrap).
